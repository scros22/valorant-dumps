## Safety verification

After the run completed, both VALORANT (PID 21184, 172 threads) and `vgc` were still running unchanged. The game received zero writes — only kernel-mode physical-memory reads of executable section pages. Vanguard cannot detect this path because it doesn't show up as a user-mode handle, an injected module, an IAT modification, or any anomalous code execution within VALORANT itself.

## Reproducibility

To re-run on any future game patch:

```
.\build\Release\vdumper.exe --valorant
```

If a previously-successful sig fails after a Riot patch, add fallback patterns to `kTier1` (or `kTier2` for brittle ones) in `src/research/valorant_automation.hpp` and rebuild with `build.bat`. The scanner is deterministic; offsets above will reproduce byte-exactly until Riot patches the binary.
